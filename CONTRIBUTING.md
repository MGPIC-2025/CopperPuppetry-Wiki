# 贡献指南

感谢你对 Copper Puppetry Wiki 的关注！我们欢迎所有形式的贡献。

## 🤝 如何贡献

### 内容贡献

你可以贡献以下内容：
- 📝 添加或更新游戏攻略
- 🤖 补充英雄详细信息
- ⚔️ 完善装备说明
- 🏗️ 增加建筑策略
- 🐛 修复错误和笔误
- 🎨 改进页面设计

### 贡献流程

1. **Fork 仓库**
   - 点击页面右上角的 Fork 按钮
   - 克隆你的 Fork 到本地

2. **创建分支**
   ```bash
   git checkout -b feature/add-hero-bubble
   ```

3. **进行修改**
   - 编辑 Markdown 文件
   - 遵循现有的格式和风格

4. **本地测试**
   ```bash
   npm run docs:dev
   ```

5. **提交更改**
   ```bash
   git add .
   git commit -m "添加 Bubble 英雄详细攻略"
   git push origin feature/add-hero-bubble
   ```

6. **创建 Pull Request**
   - 在 GitHub 上创建 PR
   - 描述你的修改内容
   - 等待审核

## 📝 内容规范

### Markdown 格式

#### 标题层级
```markdown
# 一级标题（页面标题，每页只有一个）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（小节）
```

#### 强调文本
```markdown
**粗体** - 重要内容
*斜体* - 强调
`代码` - 术语、数值、代码
```

#### 列表
```markdown
- 无序列表项 1
- 无序列表项 2

1. 有序列表项 1
2. 有序列表项 2
```

#### 表格
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
```

#### 提示框
```markdown
::: tip 提示标题
这是一条提示信息
:::

::: warning 警告标题
这是一条警告信息
:::

::: danger 危险标题
这是一条危险警告
:::

::: info 信息标题
这是一条普通信息
:::
```

### 内容风格

#### 语言风格
- ✅ 使用简洁明了的语言
- ✅ 保持友好和热情的语气
- ✅ 使用第二人称（你）而非第三人称
- ❌ 避免过于专业的术语
- ❌ 不使用脏话或不当言语

#### 格式一致性
- 数值用 ` ` 包裹：`100 点攻击力`
- 英雄名用 ` ` 包裹：`Spark`
- 装备名用 ` ` 包裹：`棱镜 (Prism)`
- emoji 适度使用，不要过多

#### 内容结构

## 🎨 样式指南

### 颜色使用

主题色：
- 主色：`#f59e0b`（橙铜色）
- 辅色：`#06b6d4`（青色）
- 文字：白色/浅灰

### 图标使用

推荐使用的 Emoji：
- 游戏相关：🎮 🎯 🎲 🎴 ⚔️ 🛡️ 🏹 🔥 ⚡ ✨
- 职业：✨（奥术师）🔨（工匠）🛡️（铁壁）⚙️（机械师）🎵（共鸣者）
- 功能：📝 💰 🏗️ 💎 ⭐ 🎯 📊 💡 🌟 📚
- 状态：✅ ❌ ⚠️ 💡 🔥

### 特殊样式块

使用 HTML 创建特殊样式的信息块：

```markdown
<div style="border: 2px solid #f59e0b; padding: 1.5rem; border-radius: 12px; background: rgba(245, 158, 11, 0.08); margin: 1.5rem 0;">
  <h2>标题</h2>
  <p>内容</p>
</div>
```

## ✅ Pull Request 指南

### PR 标题

使用清晰的标题：
- ✅ `添加：Bubble 英雄详细攻略`
- ✅ `修复：Spark 技能描述错误`
- ✅ `更新：装备系统说明`
- ❌ `update`
- ❌ `fix bug`

### PR 描述

包含以下信息：
```markdown
## 更改内容
- 添加了 XXX
- 修复了 XXX
- 更新了 XXX

## 更改原因
说明为什么需要这些更改

## 测试
- [ ] 本地测试通过
- [ ] 构建成功
- [ ] 链接正常

## 截图
（如果有视觉变化）
```

## 📜 代码规范

### 文件命名
- 使用小写字母和连字符：`hero-name.md`
- 英文名优先：`spark.md` 而非 `火花.md`
- 简短明了

### 目录结构
```
heroes/
├── index.md                 # 英雄总览
├── arcanist/               # 职业目录
│   ├── bubble.md
│   ├── spark.md
│   └── ...
├── craftsman/
└── ...
```

### 链接规范
- 使用相对路径：`[Spark](/heroes/arcanist/spark)`
- 确保链接有效
- 内部链接使用 `.md` 扩展名
